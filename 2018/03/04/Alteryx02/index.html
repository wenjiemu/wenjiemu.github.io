Alteryx系列之二：数据融合（Data Blending）

发表于 2018-03-04

该部分融合了《Data Blending for Dummies》以及《Learning Alteryx》的第三章。

1. 什么是数据融合

做分析前，需要接入各类数据，清洗和准备，这项工作又称为数据融合。数据融合是一个过程而非一次性事件，根据需要我们可以不断增加、删除或更新数据。数据融合与数据集成的区别如下：

  • 数据集成是IT术语，通常指整合多种来源的数据，经过清洗，在数据库、数据仓库或数据集市中建立统一的数据实体。DBA或BI专家负责接入数据，这些数据成为其他系统或工作流的数据源。
  • 数据融合由商业分析员或数据分析师操作，他们构建一个数据集为了解决具体的业务问题。

2. 如何做数据融合

数据融合包括以下内容：

  • 接入数据：选数据和拿数据
  • 清洗数据：重整格式，去伪存真
  • 连接数据：合并数据，以备后用

在此基础上，数据分析师还会做两件事情：

  • 高级分析：例如预测分析和空间分析
  • 输出结果：形象地表达观点，或为其他处理过程提供数据

2.1. 数据准备

数据清洗和准备阶段往往占据60%-80%的时间。数据分析师从广泛的数据来源中筛选哪些需要哪些不需要，并检查数据的质量和完整性，在此基础之上，数据分析师往往会对数据进行转换和重整。其中的关键环节如下：

  • 重构、重排、修复数据
  • 找到缺失和错误数据并更正
  • 剔除大量的无关数据
  • 明确数据集的规模，全量或部分

2.2. 数据连接

当获得远离错误，格式可用，主题相关的数据后，分析师需要把这些分散的数据连接在一起并创建数据集以备后用。Alteryx在数据连接方面可以提供以下功能：

  • 连接任意数据类型
  • 实施行级连接
  • 多字段连接
  • 模糊逻辑匹配(Fuzzy logic matching)
  • 放到地理空间中考虑问题

逻辑模糊匹配，例如从销售信息和地址信息中创建客户档案，这些信息中的名字可能不完全一致，存在各种缩写。逻辑模糊匹配将一个地址上的John, Jon, Jonny, Johnathan看成是一个人。

2.3. 自动化和再现性

现实中经常需要定期重跑分析流程，Alteryx依靠其工作流处理过程来支持该自动化需求，具体如下：

  • 创建宏，以备复用
  • 支持定制分析过程
  • 新数据自动触发运行已有处理过程
  • 处理过程保存为应用apps

3. 用Alteryx做数据融合

用好Alteryx，可以最大化数据融合的价值，具体有以下措施：

  • 用好预测分析，好的数据融合有利于生成更精确的预测模型
  • 利用APIs和SDK编程
  • 用好空间分析
  • 利用分析应用Apps，在Alteryx Analytics Gallery上找到有用的Apps，下载运行
  • 与数据可视化相结合（市面的可视化软件大多不擅长高级数据融合，并缺乏预测分析和空间分析的能力）

3.1 数据准备

  • Auto Field：自动调整元数据，将每个字符串字段都自动更新为最小的字段类型和大小
  • Unique：唯一值和重复值筛选
  • Imputation：插值
  • Multi-Field Binning：数据桶，等记录或等距

3.2 数据清洗

  • 去掉首尾空白
  • Tabs, Line Breaks, and Duplicate Whitespace
  • Punctuation removes the following characters:! " # $ % & ’ ( ) * + ,  - . / : ; < = > ? @ [ / ] ^ _ ` { | } ~

3.3 数据过滤

  • Select：选择或取消选择字段，重命名字段，更改数据类型和大小，为字段添加说明
  • 自定义过滤器： 添加表达式

3.4 Join and union

  • Join
  • Join multiple：此外，可以用自定义过滤器按照组合规则筛选
  • Union：此外，可以将几个工具组合成工具容器，并禁用该容器
